# Elastos Supernode Monitor

A lightweight dashboard that shows you exactly how much of your server your Elastos supernode is actually using. Beautiful charts, dead simple.

**Tracks**: ELA, ESC, ESC-Oracle, EID, EID-Oracle, ECO, ECO-Oracle, PG, PG-Oracle, Arbiter

**Zero native dependencies** — never fails due to missing compilers.

---

## Quick Install (One Command)

SSH into your supernode server, then:

```bash
cd /opt && sudo git clone https://github.com/4HM3DMD/elastos-node-monitor.git && cd elastos-node-monitor && bash install.sh
```

Or if you already have the files on the server:

```bash
cd /path/to/elastos-node-monitor && bash install.sh
```

That's it. The script handles everything and prints your dashboard URL at the end.

---

## Step-by-Step Install (Explain Like I'm 5)

### Step 1: Get into your server

Open your terminal and SSH into your supernode:

```bash
ssh root@YOUR_SERVER_IP
```

(Replace `YOUR_SERVER_IP` with your actual server IP address, like `145.23.67.89`)

### Step 2: Get the files onto your server

Copy the `elastos-node-monitor` folder to your server. You can use `scp` from your local machine:

```bash
scp -r ./elastos-node-monitor root@YOUR_SERVER_IP:/opt/
```

Or if you have git on the server:

```bash
cd /opt
git clone https://github.com/4HM3DMD/elastos-node-monitor.git
```

### Step 3: Run the installer

```bash
cd /opt/elastos-node-monitor
bash install.sh
```

The installer will:
- Check if Node.js is installed (installs it if not)
- Install 2 small JavaScript packages
- Figure out your CPU, RAM, and disk automatically
- Find your Elastos node directory automatically
- Create a secret access token
- Set everything up to run in the background forever
- Print your dashboard URL

### Step 4: Open the dashboard

The installer prints a URL at the end. It looks like this:

```
http://145.23.67.89:9999?token=abc123yoursecrettoken
```

Copy that URL and paste it in your browser. **Bookmark it** — it has your secret token in it.

### Step 5: Wait 2 minutes

The first charts will be empty. Give it 2 minutes to collect some data points, then refresh. After 24 hours you'll have beautiful trend lines.

## What You See on the Dashboard

### Big Gauges at the Top
Three big colored circles showing:
- **CPU** — how much brain power your server is using (green = barely using it, red = working hard)
- **RAM** — how much memory is in use
- **Disk** — how full your hard drive is

### RAM per Chain / Disk per Chain
Two bar charts showing how much RAM and disk each Elastos chain is using, always in GB. You can see which chain is the hungriest at a glance.

### Over Time
Line charts that show usage over the last hour, day, week, or month. You can see if things are getting worse or staying flat.

### The Verdict
A summary that tells you:
- What your peak and average usage actually is
- What size server you actually need (2x CPU headroom for consensus spikes, 1.5x RAM for sync bursts, 2-year disk growth projection, with Elastos official minimums enforced as floor)
- How much money you could save if you downsized

## Useful Commands

```bash
# Check if the monitor is running
sudo systemctl status elastos-monitor

# Watch the logs in real time
sudo journalctl -u elastos-monitor -f

# Restart the monitor
sudo systemctl restart elastos-monitor

# Stop the monitor
sudo systemctl stop elastos-monitor

# Completely remove the monitor
sudo systemctl disable --now elastos-monitor
sudo rm /etc/systemd/system/elastos-monitor.service
rm -rf /opt/elastos-node-monitor
```

## Configuration

The `.env` file is auto-generated by the installer. You rarely need to touch it.

| Variable | What it does | Default |
|---|---|---|
| `PORT` | Which port the dashboard runs on | `9999` |
| `AUTH_TOKEN` | Secret token for access | Auto-generated |
| `ELASTOS_NODE_DIR` | Where your Elastos node lives | Auto-detected |
| `DISK_MOUNT` | Which disk partition to monitor | `/` or `/data` |
| `CPU_CORES` | Override detected CPU cores | Auto-detected |
| `RAM_GB` | Override detected RAM | Auto-detected |
| `DISK_GB` | Override detected disk size | Auto-detected |

## How It Works (Technical)

- Collects system metrics every 30 seconds by reading `/proc/stat`, `/proc/meminfo`, `/proc/net/dev` and running `df` and `ps`
- Finds Elastos processes by matching binary names (`ela`, `esc`, `eid`, `eco`, `pg`, `arbiter`) and oracle scripts (`crosschain_*.js`) from `ps` output
- Per-process CPU from `/proc/$PID/stat` deltas (real-time, not lifetime average), RAM from `/proc/$PID/status` VmRSS, disk from `du` on chain directories
- Stores data in simple JSONL files (one per day) in the `data/` directory
- Auto-deletes data older than 30 days
- Serves a single HTML dashboard with Chart.js — no build step, no React, no webpack
- Total npm dependencies: `express` and `dotenv` (pure JavaScript, no native code)
- Uses ~15 MB of RAM and negligible CPU

## Firewall Note

The dashboard runs on port `9999`. If you can't access it from your browser, you may need to open that port:

```bash
# UFW (Ubuntu)
sudo ufw allow 9999/tcp

# Or iptables
sudo iptables -A INPUT -p tcp --dport 9999 -j ACCEPT
```

The dashboard is protected by your auth token, but if you want extra security, you can access it through an SSH tunnel instead:

```bash
# From your local machine — no need to open port 9999
ssh -L 9999:localhost:9999 root@YOUR_SERVER_IP
# Then open http://localhost:9999?token=YOUR_TOKEN in your browser
```
